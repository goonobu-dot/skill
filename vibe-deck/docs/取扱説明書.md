# Vibe Deck for D200X 取扱説明書

OpenAI Codex Micro に近い「エージェントの色監視＋手元操作」を、Ulanzi D200X 上で Claude Code / Codex / Cursor 向けに実現するセットです。

## 1. 同梱物

| 項目 | 内容 |
|------|------|
| Bridge | エージェント状態を集約する常駐プロセス（高速キャッシュ＋監視） |
| Plugin | 色タイル更新・プロファイル巡回・ページ切替 |
| Presets | `Vibe · Claude Code` / `Vibe · Cursor` / 公式 `Codex_D200X` |
| 本説明書 | セットアップと使い方 |
| クイックリファレンス | キー配置の一覧 |

## 2. 必要なもの

- Ulanzi D200X（USB 接続）
- Ulanzi Studio 3.1+（Apple Silicon）
- macOS 12+
- 対象アプリ: Claude.app / ChatGPT.app（Codex）/ Cursor.app
- Node.js 20+
- Python 3 + Pillow（アイコン生成）

## 3. セットアップ

```bash
cd /path/to/skill/vibe-deck
./scripts/install.sh
```

その後:

1. **Ulanzi Studio を完全終了→再起動**
2. プロファイルに `Vibe · Claude Code` / `Vibe · Cursor` / `Codex_D200X` があること
3. **システム設定 → プライバシーとセキュリティ → アクセシビリティ** で Ulanzi Studio を許可
4. 同じ項目で **自動化** も許可（プロファイル切替の補助に使います）

ナビだけ再配線する場合:

```bash
python3 scripts/generate-icons.py
python3 scripts/generate-tool-themes.py
node scripts/wire-deck.mjs
# その後 Studio 再起動
```

## 4. 本体操作（重要）

### 下段の左右物理ボタン = プロファイル巡回

- 左（`0_3`）: 前のプロファイル
- 右（`1_3`）: 次のプロファイル
- 対象は Studio に存在する **全プロファイル**（いま 6 個なら 6、5 個なら 5）
- 実装: キー押下でヘルパーアプリを起動 → `CurrentProfile` を更新 → Studio に反映（UI選択、だめなら短い再起動）
- 初回は **システム設定 → プライバシーとセキュリティ → アクセシビリティ** で `osascript` / ターミナル / Ulanzi Studio を許可すると切替がスムーズです
- 押すと短い効果音が鳴ります。切り替わらないときは `~/Library/Logs/vibe-deck-profile-switch.log` を確認

### 右端ダイヤル = ページ切替

- 右端ダイヤル（`4_3`）を左右に回すと、**今のプロファイル内**で Page 1 ↔ 2 ↔ 3
- ページが 2 つしかないプロファイルは 1↔2 のみ

### AI プロファイルのページ構成

| ページ | 内容 |
|--------|------|
| 1 Control | エージェント色タイル＋基本操作 |
| 2 Skills | 定型プロンプト／スキル相当ショートカット |
| 3 More | 設定・Help・Refresh など |

## 5. 色とアイコン（Codex Micro 互換）

ステータス色は全ツール共通です。

| 見た目 | 状態 | ラベル（英語のみ） |
|--------|------|---------------------|
| 白＋一時停止 | idle | Idle |
| 青＋軌道 | thinking | Thinking |
| 緑＋チェック | done | Done |
| 橙＋吹き出し? | needs_input | Input |
| 赤＋警告 | error | Error |
| 灰 | ready lane | Ready（未使用レーン。押すとそのツールを前面化） |

Studio のタイトル文字はオフにし、ラベルはアイコン内の英語のみ表示します（日英の重なり防止）。

### ツール別デザイン

コマンドキーはツールごとに見た目が違います。

| プロファイル | トーン |
|--------------|--------|
| Codex_D200X | 黒＋ティール（チャット） |
| Vibe · Cursor | ダーク IDE＋青（キャレット） |
| Vibe · Claude Code | 深紫＋オレンジ（スパーク） |

## 6. 色のレスポンス

プラグインは約 150ms 間隔で差分更新します。ブリッジは Cursor の transcript を監視し、短いキャッシュで応答します。  
体感の上限は Cursor 側が transcript を書き出すタイミングです。

## 7. ブリッジ

- URL: `http://127.0.0.1:17823`
- `GET /health`
- `GET /status?tool=codex|claude|cursor`
- `GET /status?tool=cursor&fresh=1` … キャッシュ無視

## 8. トラブルシュート

**色が遅い／変わらない**

1. `curl http://127.0.0.1:17823/health`
2. Studio 再起動（プラグイン再読込）
3. `launchctl kickstart -k gui/$(id -u)/com.vibe.deck.bridge`（LaunchAgent 利用時）

**プロファイルが切り替わらない**

1. アクセシビリティ／自動化で Ulanzi Studio を許可
2. 下段キーが「前PF／次PF」表示か確認
3. `node scripts/wire-deck.mjs` 再実行→ Studio 再起動

**ページが切り替わらない**

1. 右端ダイヤルが「Pages」になっているか
2. 対象プロファイルに Page が 2 つ以上あるか
3. Studio 再起動後に再試行

## 9. アンインストール

```bash
launchctl unload ~/Library/LaunchAgents/com.vibe.deck.bridge.plist 2>/dev/null || true
rm -f ~/Library/LaunchAgents/com.vibe.deck.bridge.plist
rm -rf ~/Library/Application\ Support/Ulanzi/UlanziDeck/Plugins/com.vibe.deck.status.ulanziPlugin
pkill -f vibe-deck/bridge/dist/index.js || true
```
