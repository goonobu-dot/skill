# 要件整理 — 質問バンク

次の判断に必要な分だけ聞く。**1回あたり最大4問**。ユーザーが「進めて」「お任せ」なら前提を置いて進む。

## 最初に使う汎用質問（1〜3個）

- 最終的に何が欲しいか（提案書 / デザイン / 動くアプリ / 公開サイト / 自動化 / レポート / 修正）
- エンドユーザーまたは意思決定者は誰か
- 期限または希望スピード
- 既存のサイト、システム、ブランド、リポジトリ、データはあるか
- 変更・公開してはいけないものはあるか

## ホームページ依頼

- 目的（信頼、集客、予約、販売、採用、ポートフォリオ など）
- 初版のページ数
- ロゴ、写真、文案、ドメイン、ホスティング、問い合わせ先はあるか
- 訪問者に取ってほしい行動
- 参考にしたい/避けたいサイト
- 静的 / CMS / EC / 会員 / カスタムアプリ のどれに近いか

## システム開発

- どの業務を置き換え・改善するか
- ユーザーロールと権限
- 保存・インポート・エクスポート・同期するデータ
- 必要な外部連携
- 必須機能（認証、決済、通知、承認、監査ログ など）
- 社内ツール / 対外 SaaS / モバイル / API のどれか

## 自動化 / AI エージェント

- 繰り返している作業と頻度
- 入力元と出力先
- 人の承認が必要なステップ
- 失敗したときのコスト（金銭、信用）
- ログ・リトライ・手動オーバーライドの要否
- 利用可能なアカウント、API、ファイル

## 調査 / 戦略

- この調査で決めたいこと
- ソース（公式のみ / 内部 / 混合）
- 成果物の形（推奨ランキング、メモ、レポート、スライド、実装計画）
- 重視する基準（コスト、速度、信頼性、リスク など）

## 質問を止めて計画に進む条件

以下が安全に定義できれば、追加質問せず計画を書いて着手する：

- 成果物の形
- 初版スコープ（含む / 含まない）
- 主要リスクと対策
- 次の具体的アクション
